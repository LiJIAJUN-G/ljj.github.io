docker基本概念与使用教程

docker

docker基本命令的学习

基本概念

  1. 仓库: repository 远程仓库(全世界开发者镜像以及官方发镜像) 本地仓库(只保存当前自己使用过的镜像以及自己定义镜像) 用来存放docker镜像位置作用

  2. 镜像: images 作用:一个镜像就代表一个软件如: mysql redis tomcat

  3. 容器: container 作用:一个镜像运行一次就会生成一个实例|就是生成一个容器

  4. docker入门

    
docker run hello-world

docker引擎以及镜像的相关操作

查看docker引擎以及docker帮助命令


docker info用来展示docker信息(版本等等)
docker version
docker --help ll docker查看docker所有帮助命令
docker 执行命令格式: docker [options] command(具体命令)

操作镜像images的相关命令

  1. 查看本地仓库中存在哪些镜像

    
docker images

  2. 下载新的镜像

    
docker pull 镜像名称:版本号(推荐)
docker pull mvsq1:5.6.5
docker pull 镜像名:@DIGEST(摘要)
docker pull mysql:DIGEST:sha256:d8e4032005f53a774f2361281ebf61fa3d7635d5dacf9c58bc54e823ddcf9f1d

  3. 不通过dockerhub 通过命令行搜索需要镜像

    
docker search mysq

  4. 删除镜像

    
docker image rm 镜像名:tag | 镜像id
docker image rm -f 镜像名:taql镜像id 强制删除
docker rmi -f 镜像名:tag I镜像id 简化写法

docker容器基本操作

docker load -i 导入tar的镜像文件名

  • 通过镜像运行一个容器

    
docker run 镜像名:tag | 镜像id
    1. 以tomcat镜像为例运行tomcat容器(运行tomcat实例)
    
docker run tomcat:8.0-jre8
    1. 宿主机端口与容器中端口进行映射 -P
    
docker run -p 8080(系统上外部口):8080(容器内服务监听的端口) tomcat:8.0-jre8
    1. 启动容器 映射外部端口 后台启动
    
docker run -p 8080(系统上外部口):8080(容器内服务监听的口) -d(后台运行) tocat:8.0-jre8
    1. 启动容器 指定名称 后台运行 端口映射
    
docker run-d(后台运行)-p 8081:8080(书写多个) --name tomcat01(器名称)tomcat:8.0-jre8(镜像名称:tag)

  • 查看当前运行的容器

    
docker ps查看正在运行容器
docker ps -a 查看所有容器(运行 & 非运行)
docker ps -q 返回正在运行容器id
docker ps -qa 返回所有容器的id

  • 停止|关闭|重启容器

    
docker start容器名字或者容器id    开启容器
docker restart 容器名或者容器id     重启容器
docker stop容器名或者容器id   正常停止容器运行
docker kill容器名或者容器id   立即停止容器运行

  • 删除容器

    docker rm容器名称|容器id   删除停止的容器
docker rm -f 容器名称I容器id   删除正在运行的容器
docker rm-f $(docker ps -aq)   删除所有容器

  • 查看容器内服务运行日志

    docker logs容器id I 容器名称
docker logs -f 容器id I容器名称  实时展示日志
docker logs -tf 容器id I容器名称  加入时间戳实时展示日志

  • 查看容器内进程

    docker top 容器id或者容器名   查看容器内的进程

  • 与容器内部进行交互

    docker exec -it容器id或者容器名 bash(命行)
进入器与器内终进行交
Exit         退出容器

  • 操作系统与容器的传输文件

    从容器复制文件到操作系统
docker cp 容器唯一标识(id,name):容器内源路径 操作文件路径/文件目录
从操作系统复制文件到容器
docker cp文件|目录名 容器唯一标识(id,name):/容器内资源路径

  • 查看容器内部细节

    docker inspect 容器id/容器名称

  • 数据卷 volume

    作用:实现宿主机系统与容器之间的文件共享

数据卷使用

  1. 自定义数据卷目录

    
docker run -d -p 8080:8080 --name tomcat -v宿主机名录(必须绝对):容器内目录tomcat:8.0-jre8

  2. 自定义数据卷目录

    
docker run -d -p 8081:8080 --name tomcat02 -y /root/apps/:/usr/local/tomcat/webapps tomcat:8.0-jret

  3. 自动数据卷目录

    
 docker run -d -p 8083:8080 --name tomcat03 -v aa:/usr/local/tomcat/webapps tomcat:8.0-jre8

    注意: aa代表一个数据卷名字,名称可以随便写,docker在不存在时自动创建这个数据卷同时自动映射宿主机中某个目录 同时在启动容器时会将aa对应器录中全部内复制到aa射目录中

  4. 将容器打包成一个新的镜像

    
docker commit -m"描述信息”-an作者信息”(容器id或者名称)打包的镜像名称: 标签

  5. 将镜像备份出来

    
docker save镜像名称:Tag -o文件名
docker save test-tomcat:1.0 -o test-tomcat-1.0.tar
